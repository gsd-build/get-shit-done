# Решение

                         Telegram-бот с двумя режимами:  

  - Light-режим: Пошаговый опросник   

  - Expert-режим: Свободный ввод описания товара

                               Основная функция MVP:  

  - Определение кода ТН ВЭД с указанием вероятности правильности его определения и нормативного обоснования выбранного при соответствующем запросе   

Проблема 

Сложность определения кода ТН ВЭД в связи со спецификой отношений

# Критерии приемки MVP

1. Функциональные: все команды работают, два режима работы бота 

- /start – приветственное сообщение и выбор режима (Light/Expert).
- /help – справка с описанием команд.
- /classify – запуск процесса классификации (в текущем режиме).
- /check – проверка существующего кода.
- /history – просмотр истории запросов.
- /cancel – завершение текущего диалога и очистка введённых данных

2. Технические: безопасность, логирование  

3. Пользовательские: понятный интерфейс, юридически корректные формулировки

# Техническое задание - суть

Бот реализуется на платформе Telegram. 

Функционально доступны 2 режима:

- Light‑режим (опросник) – бот задаёт уточняющие вопросы по готовой анкете; вопросы подобраны таким образом, чтобы минимизировать количество шагов и повышать точность определения кода.
- Expert‑режим – пользователь сразу предоставляет максимально полное описание товара (характеристики, назначение, материал, изображение). Алгоритм обрабатывает введённый текст и возвращает коды без подробных уточнений. Этот режим подходит опытным участникам ВЭД.

Бот использует актуальную редакцию Единой товарной номенклатуры внешнеэкономической деятельности (ТН ВЭД) Евразийского экономического союза (ЕАЭС), которая включает 21 раздел и 97 групп товаров.

# Техническое задание - суть

Управление режимом и завершение диалога. Пользователь в любой момент может сменить режим или завершить диалог. Команды:

- /start – Приветствие, выбор режима (Light или Expert), краткое описание возможностей бота 
- /help – Отображение списка команд и разъяснение назначения каждой.
- /classify – Запуск процесса классификации (в текущем режиме). В режиме Light запускает опросник; в режиме Expert предлагает ввести описание товара
- /check – Проверка имеющийся код ТН ВЭД - бот возвращает официальное описание, актуальность и предупреждение о возможных изменениях.
- /history – просмотр истории запросов.
- /cancel – завершение текущего диалога и очистка введённых данных.

# Техническое задание - суть

Функциональные критерии

1. Пользователь может выбрать режим (Light/Expert), ввести описание товара и получить список предложенных кодов ТН ВЭД с оценкой вероятности.
2. В режиме Light бот задаёт уточняющие вопросы, поддерживает возможность досрочного завершения сессии командой /cancel.
3. В режиме Expert бот корректно обрабатывает свободный текст.
4. Команда /check проверяет имеющийся код и сообщает об актуальности
5. Команды /start, /help, /classify, /history, /cancel работают без ошибок; история запросов отображается корректно.

# Нормативная база

- Основной НПА, применимый к проекту - Решение Совета Евразийской экономической комиссии от 14.09.2021 N 80 "Об утверждении единой Товарной номенклатуры внешнеэкономической деятельности Евразийского экономического союза и Единого таможенного тарифа Евразийского экономического союза, а также об изменении и признании утратившими силу некоторых решений Совета Евразийской экономической комиссии" 
- В данном НПА содержатся актуальные коды ТН ВЭД, для определения которых предназначен наш чат-бот.

# Группы товаров, на которых специализируется MVP

- 8408 Двигатели внутреннего сгорания поршневые с воспламенением от сжатия (дизели или полудизели)
- 8418 Холодильники, морозильники и прочее холодильное или морозильное оборудование электрическое или других типов; тепловые насосы, кроме установок для кондиционирования воздуха товарной позиции 8415

# Алгоритм действий бота

Алгоритм является приложением к презентации. 

Задача по алгоритму, решаемая на данный момент:

-  В алгоритм планируется добавить следующее правило: «Если пользователь выбрал режим-Expert, но при этом данных, введенных им в качестве описания товара, недостаточно для достоверного определения кода ТН ВЭД, бот предлагает пользователю сменить режим на Light». 

На данный момент предстоит разработать четкие критерии определения достаточности введенных пользователем данных в режиме Expert.

# Проблемы с тг ботом

Любое внесение изменений в код порождает баги. 

По ходу изменения режимов light и expert надо перерабатывать механику, в частности команду /classify.

Любые непрописанные команды, баги в коде - баги в боте, т.к. бот работает исключительно на запущенном программном коде.

Сейчас задачи:

1. Встроить в бота нейронку (ChatGPT Plus) 
2. Сделать режим Light в формате диалога, т.к. на данный момент он выполнял функцию проверки имеющегося кода (по сути команду /check)
3. Режим Expert только для подробных технических данных – в случае ввода недостаточных данных бот должен предлагать режим Light
4. Разработать критерии определения достаточности введенных пользователем данных в режиме Expert для точного определения кода ТН ВЭД
