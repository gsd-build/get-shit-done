## Directory Purposes

**bin/:**
- Purpose: Installation scripts
- Contains: Entry point for `npx get-shit-done-cc` command
- Key files: `install.js` (handles --claude, --opencode, --gemini, --global, --local flags)

**agents/:**
- Purpose: Specialized subagent implementations
- Contains: 11+ Markdown files, each a complete agent role
- Key pattern: Each agent has role, context requirements, execution flow, output format
- Naming: `gsd-<role>.md` (e.g., gsd-planner.md, gsd-executor.md)

**commands/gsd/:**
- Purpose: User-facing command registry
- Contains: 28+ commands covering project lifecycle, phase workflow, system operations
- Naming: `<action>.md` (e.g., new-project.md, plan-phase.md)
- Frontmatter: name, description, agent (which agent to spawn), allowed-tools

**get-shit-done/references/:**
- Purpose: Shared knowledge consumed by agents
- Contains: Pattern libraries, checklists, decision tables
- Key files:
  - `questioning.md` — 11 questioning techniques
  - `verification-patterns.md` — How to verify different types of work
  - `model-profiles.md` — Model assignments per agent per profile
  - `planning-config.md` — Config.json option reference

**get-shit-done/templates/:**
- Purpose: Markdown templates for generated documents
- Contains: 30+ templates for different document types
- Usage: Agents and orchestrators use these as templates for generated PROJECT.md, PLAN.md, etc.
- Subdirectories:
  - `codebase/` — Templates for codebase analysis (ARCHITECTURE.md, CONVENTIONS.md, etc.)
  - `research-project/` — Templates for research output (SUMMARY.md, FEATURES.md, etc.)

**get-shit-done/workflows/:**
- Purpose: High-level orchestration workflows
- Contains: Markdown describing multi-agent coordination
- Key files:
  - `execute-phase.md` — Wave-based parallel execution orchestration
  - `map-codebase.md` — Codebase analysis workflow (spawns 4 parallel mappers)
  - `verify-work.md` — Verification workflow

**hooks/:**
- Purpose: Integration with Claude Code and git
- Contains: JavaScript hooks and compiled versions
- Key file: `gsd-statusline.js` — Shows GSD task + context usage in Claude Code statusline

**scripts/:**
- Purpose: Build and utility scripts
- Key file: `build-hooks.js` — Compiles hooks from source

**.planning/:**
- Purpose: Project-specific planning artifacts (generated by GSD commands)
- Structure:
  - `codebase/` — When using `/gsd:map-codebase` on existing projects
  - `phases/` — When executing phases
  - Root level: PROJECT.md, ROADMAP.md, REQUIREMENTS.md, STATE.md, config.json

**.claude/:**
- Purpose: Local Claude Code configuration
- Contains: `.claude/get-shit-done/` (local GSD installation), settings.json, rules
- Structure mirrors global `~/.claude/` but project-specific

## Naming Conventions

**Files:**
- Agent files: `gsd-<role>.md` (lowercase, hyphens, no periods)
- Command files: `<action>.md` (lowercase, single action verb)
- Markdown documents: `UPPERCASE.md` in user projects
- Configuration: `config.json`, `settings.json`
- JavaScript: `<name>.js` (source in hooks/, compiled in hooks/dist/)

**Directories:**
- Resources: lowercase, hyphens (`get-shit-done/`, `.planning/`, `commands/gsd/`)
- Phases: zero-padded numbers + dash + name (`01-setup`, `02-auth`, `10-deployment`)
- Archives: `_archive/` subdirectories (for deprecated files)

**Planning Documents:**
- Global: `PROJECT.md`, `ROADMAP.md`, `REQUIREMENTS.md`, `STATE.md`, `config.json`
- Per-phase: `CONTEXT.md`, `RESEARCH.md`, `PLAN.md`, `SUMMARY.md`, `VERIFICATION.md`

## Where to Add New Code

**New Agent:**
1. Create `agents/gsd-<role>.md`
2. Structure: role definition → context requirements → execution flow
3. Register by adding to orchestrator command that spawns it (via Task tool)
4. Example: Look at `agents/gsd-executor.md` for pattern

**New Command:**
1. Create `commands/gsd/<action>.md`
2. Frontmatter: name, description, agent, allowed-tools
3. Content: execution_context (@file references) → process (steps) → output
4. If command orchestrates multiple agents: Put workflow in `get-shit-done/workflows/`
5. Example: Look at `commands/gsd/plan-phase.md` for orchestrator pattern

**New Reference Document:**
1. Add to `get-shit-done/references/<topic>.md`
2. Include CLAUDE.md header for agent navigation
3. Consumed by agents via @ references (e.g., `@~/.claude/get-shit-done/references/questioning.md`)

**New Template:**
1. Add to `get-shit-done/templates/<document>.md`
2. Include frontmatter with description
3. Placeholder sections marked with `[PLACEHOLDER]` or `[DATE]`
4. Used by generators via template filling

**Utilities for User Projects:**
1. If utility is domain-specific (e.g., TypeScript linting): Store in `get-shit-done/scripts/`
2. If utility is general: Could go in hooks or scripts
3. Update `scripts/build-hooks.js` if hook needs compilation

## Special Directories

**.planning/ (generated, project-specific):**
- Purpose: Store all project artifacts (questionnaire, plans, state)
- Generated by: `/gsd:new-project` (creates root files), `/gsd:plan-phase` (creates phase directories)
- Committed: Yes (unless `commit_docs: false` in config or `.planning/` is gitignored)
- Structure:
  ```
  .planning/
  ├── PROJECT.md              # Project vision
  ├── ROADMAP.md              # Phase structure
  ├── REQUIREMENTS.md         # Traceability
  ├── STATE.md                # Execution state
  ├── config.json             # Settings
  ├── codebase/               # Codebase analysis docs (from /gsd:map-codebase)
  ├── phases/                 # Phase-specific artifacts
  │   ├── 01-name/
  │   │   ├── CONTEXT.md
  │   │   ├── RESEARCH.md (optional)
  │   │   ├── PLAN.md
  │   │   └── SUMMARY.md
  │   └── ...
  └── research/               # Project research (from /gsd:new-project)
  ```

**hooks/dist/ (compiled, generated):**
- Purpose: Built hooks for Claude Code integration
- Generated by: `npm run build:hooks` (runs `scripts/build-hooks.js`)
- Committed: Yes (so installation gets pre-built versions)
- Build process: Copies and minifies from hooks/*.js

**.claude/ (project-specific local installation):**
- Purpose: Local GSD installation (doesn't affect global `~/.claude/`)
- Created by: `/gsd:new-project` if using `--local` install mode
- Structure: Mirrors global installation but scoped to project
- Usage: Commands available only in this project

**agents/ (source, immutable):**
- Purpose: Agent implementations
- Committed: Yes (part of GSD codebase)
- Installed: Copies to user's `~/.claude/agents/`, `~/.opencode/`, `~/.gemini/`

**commands/gsd/ (source, immutable):**
- Purpose: Command definitions
- Committed: Yes (part of GSD codebase)
- Installed: Copies to user's `~/.claude/commands/gsd/`

**get-shit-done/ (source, immutable):**
- Purpose: Shared resources (references, templates, workflows)
- Committed: Yes (part of GSD codebase)
- Installed: Copies to user's `~/.claude/get-shit-done/`
