| Dynamic pattern regex -> process name extraction | The `sme.processes.{name}.block_mode` regex constrains process names to `[a-zA-Z0-9_-]+` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02 | Tampering | `sme.processes.{name}.block_mode` dynamic config key | mitigate | Regex `^sme\.processes\.[a-zA-Z0-9_-]+\.block_mode$` restricts process names to alphanumeric + hyphen + underscore. No path traversal characters (`.`, `/`, `\`) allowed in process name segment. Already enforced by `isValidConfigKey` at config-set entry point. |
| T-01-03 | Information Disclosure | Config defaults expose SME feature existence | accept | Config keys are developer-facing, not user-facing. Default `false` means no SME behavior is active. No sensitive data exposed. |
</threat_model>
