  <action>
Create `get-shit-done/templates/sme.md` with the following exact structure.

**YAML Frontmatter** (per D-04 -- five fields, nested finding_counts):
```yaml
---
process_name: [PROCESS_NAME]
last_analyzed_commit: [COMMIT_HASH]
block_mode: soft
created_date: [DATE]
finding_counts:
  blocker: 0
  warning: 0
  watch: 0
---
```

Use 2-space indentation for the nested `finding_counts` object (required by `frontmatter.cjs` `reconstructFrontmatter`). `block_mode` defaults to `soft`. Placeholder tokens use `[BRACKET_CAPS]` style.

**Document Title:**
```markdown
# [PROCESS_NAME] SME Document
```

**Six H2 Sections in Fixed Order** (per D-03):

1. `## Process Overview` -- prose section, no findings format. Include a placeholder paragraph:
```
[Prose description of this process: what it does, who uses it, key invariants,
and the main code paths involved. Written by the creator agent after analysis.]
```

2. `## Identified Risks` -- HTML-commented example finding using BLOCKER severity
3. `## Test Gaps` -- HTML-commented example finding using WARNING severity
4. `## Outdated Logic` -- HTML-commented example finding using WATCH severity
5. `## Edge Cases` -- HTML-commented example finding using WARNING severity
6. `## Known Blockers` -- HTML-commented example finding using BLOCKER severity

**Finding Format** (per D-01, D-02, D-08 -- all four fields required):
Each HTML-commented example must demonstrate the complete required format with severity tag, bold title, evidence with file:line reference, and mitigation. Example:
```html
<!-- [BLOCKER] **Example risk title** — Brief description of the risk.
  *Evidence:* `src/path/to/file.ts:LINE` — what makes this a risk
  *Mitigation:* Specific action to address it -->
```

Vary the severity tags and example content across sections to demonstrate the full range (BLOCKER, WARNING, WATCH). The evidence line must always include a backtick-wrapped `file:line` reference. The mitigation line must always include a concrete action, not vague guidance.

Do NOT include any real findings -- only HTML-commented examples that guide the creator agent.
