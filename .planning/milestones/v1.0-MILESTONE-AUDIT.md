### Non-Critical Findings (3)

**MC-01 — SME agents not in MODEL_PROFILES**

`model-profiles.cjs` and `config-query.ts` define MODEL_PROFILES with 17 agents — none include `gsd-sme-creator`, `gsd-sme-auditor`, or `gsd-sme-creator-analyzer`. `resolve-model` returns fallback (`sonnet`) for these agents. Workflows handle this via `|| echo "inherit"` or `|| echo "sonnet"` — no crashes, but users on `quality` profile will silently get sonnet for SME agents.

Affected: CREATE-01, AUDIT-01, GATE-03, DETECT-04, REFRESH-02

**MC-02 — SDK CONFIG_DEFAULTS missing sme sub-object**

`sdk/src/config.ts` `CONFIG_DEFAULTS` includes `workflow.use_sme_agents: false` but lacks the `sme: { blocking: 'soft' }` section that `get-shit-done/bin/lib/config.cjs` provides in `buildNewProjectConfig()`. SDK-side `mergeDefaults` will carry through a `sme` key from project `config.json` if present, but projects created before this milestone may not have it. `config-get sme.blocking` falls back to `|| echo "soft"` in the creator agent.

Affected: CONFIG-03

**MC-03 — discuss-phase.md gating incomplete vs. sme-step.md contract**

`discuss-phase.md` line 280 only checks `use_sme_agents: true` before loading sme-step.md. The sme-step.md header documents it should only load when `active_smes` is also non-empty. The sme-step.md recovers internally (Step 2 returns early if `ACTIVE_SMES` is empty). This is a documentation/contract inconsistency, not a behavioral break.

Affected: DISCUSS-01
