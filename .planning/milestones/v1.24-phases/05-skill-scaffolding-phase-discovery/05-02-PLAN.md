**Structure:**

**`<purpose>`** (1-3 sentences): Drive all remaining milestone phases autonomously. For each incomplete phase: discuss → plan → execute. Pause only for explicit user decisions.

**`<required_reading>`**: Standard — read all files referenced by execution_context before starting.

**`<process>`** with these named steps:

**Step 1: `initialize` (priority="first")**
- Parse `$ARGUMENTS` for `--from N` flag (extract N if present, default to null)
- Bootstrap via `INIT=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" init milestone-op)`
- Parse JSON for: `milestone_version`, `milestone_name`, `phase_count`, `completed_phases`, `roadmap_exists`, `state_exists`, `commit_docs`
- Error if `roadmap_exists` is false: "No ROADMAP.md found. Run `/gsd:new-milestone` first."
- Error if `state_exists` is false: "No STATE.md found."
- Display startup banner: `━━━ GSD ► AUTONOMOUS ━━━` with milestone name and version

**Step 2: `discover_phases`**
- Run `ROADMAP=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" roadmap analyze)`
- Parse JSON `phases` array
- Filter to incomplete phases: where `disk_status !== "complete"` OR `roadmap_complete === false`
- If `--from N` was provided, additionally filter out phases where `number < N` (use numeric comparison, handling decimal phases like "5.1")
- Sort by `number` (numeric order)
- If no incomplete phases remain: Display "All phases complete! 🎉" and exit
- Display phase plan: list each incomplete phase with number and name
- For each phase, fetch details: `DETAIL=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" roadmap get-phase ${PHASE_NUM})` — extract `phase_name`, `goal`, `success_criteria`

**Step 3: `execute_phase`**
- For the current phase, display progress banner: `GSD ► AUTONOMOUS ▸ Phase N/T: [Name] [████░░░░] P%` where N is current phase position, T is total phases, P is percent complete. Use ━ for the banner border and █/░ for the progress bar.
- Run discuss: `Skill(skill="gsd:discuss-phase", args="${PHASE_NUM}")`
- Verify discuss produced output: `node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" init phase-op ${PHASE_NUM}` — check `has_context`
- If discuss failed (no context), ask user: "Discuss phase failed. Fix and retry / Skip this phase / Stop autonomous mode"
- Run plan: `Skill(skill="gsd:plan-phase", args="${PHASE_NUM}")`
- Verify plan produced output: check `has_plans` from phase-op init
- If plan failed, ask user same 3 options
- Run execute: `Skill(skill="gsd:execute-phase", args="${PHASE_NUM}")`
- Display transition: "Phase N ✅ [Name] → Phase N+1: [Next Name] — [1-line goal]"

**Step 4: `iterate`**
- After each phase completes, re-read ROADMAP.md: run `roadmap analyze` again (catches decimal phases inserted mid-execution)
- Re-filter incomplete phases (same logic as discover_phases)
- Read STATE.md fresh: `cat .planning/STATE.md` — check for blockers
- If blockers found, pause and ask user
- Proceed to next incomplete phase by looping back to execute_phase
- After all phases complete, display completion banner: "━━━ GSD ► AUTONOMOUS ▸ COMPLETE ━━━" with final stats

**Step 5: `handle_blocker`**
- When any phase operation fails or blocker is detected
- Present 3 options via user question: "Fix and retry" / "Skip this phase" / "Stop autonomous mode"
- On retry: re-run the failed step
- On skip: mark phase skipped, continue to next
- On stop: display summary of progress, exit cleanly

**IMPORTANT conventions to follow:**
- Use `Skill()` for all command chaining (NOT `Task()`) — per CONTEXT.md locked decision and Issue #686
- All bash tool paths use `$HOME/.claude/get-shit-done/bin/gsd-tools.cjs` (installer rewrites)
- All `execution_context` paths use `@~/.claude/get-shit-done/` prefix
- Phase iteration is the skeleton — actual discuss/plan/execute Skill() calls are in place, but smart discuss enhancements (Phase 6), auto-chaining refinements (Phase 7), and lifecycle completion (Phase 8) will enhance these steps later
