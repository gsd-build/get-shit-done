### Directory and Path Handling
- [x] **PATH-01**: `getDirName('cursor')` returns `.cursor`
- [x] **PATH-02**: `getGlobalDir('cursor')` returns `~/.cursor/` with `CURSOR_CONFIG_DIR` env var support
- [x] **PATH-03**: Path reference replacement (`~/.claude/` → `~/.cursor/`) in file content
- [x] **PATH-04**: Command format conversion (`/gsd:` → `/gsd-`) in file content

### Installer Integration
- [x] **INST-01**: `--cursor` CLI flag recognized and adds 'cursor' to `selectedRuntimes`
- [x] **INST-02**: Cursor option in interactive `promptRuntime()` (option 4: Cursor, option 5: All)
- [x] **INST-03**: `install(isGlobal, 'cursor')` deploys files to `~/.cursor/` using nested directory structure
- [x] **INST-04**: `uninstall(isGlobal, 'cursor')` removes GSD files from `~/.cursor/`
- [x] **INST-05**: Attribution handling via `getCommitAttribution('cursor')` using settings.json pattern

## v2 Requirements (Deferred)

- Local install support for Cursor (`.cursor/` in project)
- Runtime auto-detection (detect Cursor without `--cursor` flag)
- Cursor-specific statusline enhancements

## Out of Scope

- Windows-specific PowerShell installer — covered by unified Node.js installer
- Separate cursor-gsd distribution — consolidating into main installer
- Cursor plugin/extension — GSD installs as config files, not as extension
