  <name>Task 2: Create error envelope module</name>
  <files>get-shit-done/bin/lib/mcp/errors.cjs</files>
  <action>
Create the errors module at `get-shit-done/bin/lib/mcp/errors.cjs` with:

1. `envelope(data)` - Wraps successful response:
   ```javascript
   function envelope(data, nextActions = []) {
     return {
       success: true,
       data,
       error: null,
       next_actions: nextActions,
     };
   }
   ```

2. `errorEnvelope(code, message, recovery, nextActions = [])` - Wraps error response:
   ```javascript
   function errorEnvelope(code, message, recovery, nextActions = []) {
     return {
       success: false,
       data: null,
       error: { code, message, recovery },
       next_actions: nextActions,
     };
   }
   ```

3. `GsdError` class - Custom error with recovery suggestion:
   ```javascript
   class GsdError extends Error {
     constructor(code, message, recovery, nextActions = []) {
       super(message);
       this.code = code;
       this.recovery = recovery;
       this.nextActions = nextActions;
     }
     toEnvelope() {
       return errorEnvelope(this.code, this.message, this.recovery, this.nextActions);
     }
   }
   ```

4. Pre-defined error factories for common cases:
   - `phaseNotFoundError(phase)` - Returns GsdError with code 'PHASE_NOT_FOUND'
   - `planNotFoundError(phase, plan)` - Returns GsdError with code 'PLAN_NOT_FOUND'
   - `projectNotFoundError()` - Returns GsdError with code 'PROJECT_NOT_FOUND'
   - `invalidInputError(field, reason)` - Returns GsdError with code 'INVALID_INPUT'

Each error factory should include appropriate `recovery` text referencing /gsd: commands.

Export all functions and the GsdError class.

Create the server module at `get-shit-done/bin/lib/mcp/server.cjs`:

1. Import MCP SDK:
   ```javascript
   const { McpServer } = require('@modelcontextprotocol/sdk/server/mcp.js');
   const { StdioServerTransport } = require('@modelcontextprotocol/sdk/server/stdio.js');
   ```

2. `createServer()` function:
   - Create McpServer instance with name 'gsd-mcp-server' and version from package.json
   - Return the server instance (tools/resources registered by caller)

3. `connectTransport(server)` async function:
   - Create StdioServerTransport
   - Call `await server.connect(transport)`
   - Return transport for cleanup

4. Export createServer and connectTransport

Create the entry point at `get-shit-done/bin/gsd-mcp-server.cjs`:

```javascript
#!/usr/bin/env node

const { createServer, connectTransport } = require('./lib/mcp/server.cjs');

async function main() {
  const server = createServer();

  // Tools and resources will be registered in plan 12-02 and 12-03

  await connectTransport(server);

  // Log to stderr (stdout is for JSON-RPC)
  console.error('[gsd-mcp] Server started');
}

main().catch((err) => {
  console.error('[gsd-mcp] Fatal error:', err.message);
  process.exit(1);
});
```

Make the entry point executable (chmod +x).

IMPORTANT per RESEARCH.md anti-patterns:
- Never log to stdout (interferes with JSON-RPC)
- All logs go to stderr via console.error()
