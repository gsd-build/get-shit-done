<task type="auto">
  <name>Task 1: Create GitHub Actions test workflow</name>
  <files>.github/workflows/test.yml</files>
  <action>
Create `.github/workflows/test.yml` with the following configuration:

**Workflow identity:**
- `name: Tests`

**Triggers (`on:`):**
- `push:` to `main` branch only
- `pull_request:` to `main` branch only (includes drafts by default)
- `workflow_dispatch:` for manual runs (no inputs needed)

**Concurrency:**
- Group: `${{ github.workflow }}-${{ github.head_ref || github.run_id }}`
- `cancel-in-progress: true`

This ensures same-branch/PR runs auto-cancel stale executions while push-to-main runs (no `head_ref`) get unique groups via `run_id`.

**Jobs:**
- Single job named `test`
- `runs-on: ${{ matrix.os }}`
- `timeout-minutes: 10`

**Strategy:**
- `fail-fast: true`
- Matrix:
  - `os: [ubuntu-latest, macos-latest, windows-latest]`
  - `node-version: [18, 20, 22]`

**Steps (in order):**
1. `actions/checkout@v4`
2. `actions/setup-node@v4` with:
   - `node-version: ${{ matrix.node-version }}`
   - `cache: 'npm'` (built-in npm caching)
3. `run: npm ci`
4. `run: npm test`

No artifact uploads, no extra permissions, no path filtering. Keep it minimal.
  </action>
  <verify>
