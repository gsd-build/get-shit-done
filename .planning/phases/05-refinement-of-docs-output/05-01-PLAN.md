2. Replace the flat docs/ scan block with a recursive helper function `walkDir(dir, depth)` that:
   - Returns immediately if `depth > MAX_DEPTH`
   - Reads directory entries with `fs.readdirSync(dir, { withFileTypes: true })`
   - For each entry:
     - If `SKIP_DIRS.has(entry.name)` -- skip
     - If `entry.isDirectory()` -- recurse with `walkDir(path.join(dir, entry.name), depth + 1)`
     - If `entry.isFile()` and name ends with `.md` (case-insensitive) -- add to results with `toPosixPath(path.relative(cwd, abs))` and `hasGsdMarker(abs)`
   - Wrap in try/catch for best-effort (directory may not exist)

3. Call `walkDir(docsDir, 1)` where `docsDir = path.join(cwd, 'docs')`

4. Also check for alternative docs directory names: after the `docs/` walk, check if `documentation/` or `doc/` exists (only if `docs/` did not exist). If found, walk that directory the same way. This handles edge cases per Claude's discretion from CONTEXT.md.

5. Update the JSDoc on scanExistingDocs to say "Recursively scan docs/ directory up to 4 levels deep" instead of "one level deep".

Keep the root-level .md scan (lines 53-64) exactly as-is -- no recursion at root level.
