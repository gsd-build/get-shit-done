Write `evals/sme-creator.promptfooconfig.yaml` with 10 test cases covering the 8 eval dimensions from AI-SPEC Section 5:

Top-level field: `description: SME Creator Agent -- regression eval suite`

Test 1 -- "Schema completeness -- frontmatter has last_analyzed_commit as valid git SHA":
  type: javascript assertion that reads the first *-SME.md file from .planning/smes/ and checks `/last_analyzed_commit: [0-9a-f]{7,40}/` regex match. Return false if no SME files exist.

Test 2 -- "Schema completeness -- frontmatter has block_mode field":
  type: javascript assertion that checks `/block_mode: (soft|strict)/` regex match.

Test 3 -- "Schema completeness -- frontmatter has finding_counts with blocker, warning, watch":
  type: javascript assertion that checks all three of `/finding_counts:/`, `/blocker: \d+/`, `/warning: \d+/`, `/watch: \d+/`.

Test 4 -- "Schema completeness -- document has all 6 required section headers":
  type: javascript assertion that checks sme.includes() for each of: '## Process Overview', '## Identified Risks', '## Test Gaps', '## Outdated Logic', '## Edge Cases', '## Known Blockers'.

Test 5 -- "BLOCKER severity -- every BLOCKER must cite historical evidence":
  type: llm-rubric assertion with rubric text: "Review the SME document for findings labeled [BLOCKER]. Every BLOCKER finding MUST include at least one of: a specific git commit hash (7-40 hex characters), a PR number (e.g., PR #123), or a direct quote from a code comment as evidence that the risk has a real production failure path. A BLOCKER that only describes code structure without citing historical evidence of a real production failure path is a FAIL. If no BLOCKER findings exist, PASS (absence is not a violation)."

Test 6 -- "Finding traceability -- every finding references a specific file path":
  type: javascript assertion that splits the SME content on [BLOCKER|WARNING|WATCH] markers and checks each finding block contains `File:` or a backtick-quoted file path pattern.

Test 7 -- "No hallucinated file paths -- every cited .ts path must exist in repo":
  type: javascript assertion that extracts all backtick-quoted paths matching `(src|sdk|agents|get-shit-done)` prefix patterns, then runs fs.statSync on each. Return true if all exist or if none are cited.

Test 8 -- "Process boundary -- findings should reference files matching the process name":
  type: javascript assertion that extracts process_name from frontmatter, extracts all cited file paths, checks >= 70% of cited paths contain the process name (with hyphens stripped for comparison).

Test 9 -- "Task completion -- SME document is non-empty with at least one finding":
  type: javascript assertion that checks sme.length > 200 and at least one [BLOCKER|WARNING|WATCH] marker exists.

Test 10 -- "Cleanup -- no .tmp directory left behind after agent completion":
  type: javascript assertion that checks fs.accessSync('.planning/smes/.tmp') throws (directory should NOT exist). Return true on error (cleaned up), false if accessible (not cleaned up).
