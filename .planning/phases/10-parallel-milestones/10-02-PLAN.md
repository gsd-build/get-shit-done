# Plan 10-02: Milestone-Scoped Workflow Commands

## Objective

Extend core GSD workflow commands (`plan-phase`, `execute-phase`, `verify-work`, `progress`) to support milestone-scoped operations. Users can target specific milestones with `M7/01` syntax or rely on default milestone context.

## Context

**Depends on:** Plan 10-01 (milestone directory structure and core commands)

**Commands to modify:**
- `init execute-phase` — Support `M7/01` argument
- `init plan-phase` — Support milestone context
- `roadmap analyze` — Read milestone-specific ROADMAP.md
- `progress` — Show per-milestone progress bars

## Must-Haves

### Artifacts
- Modified `get-shit-done/bin/lib/init.cjs` — Milestone-aware initialization
- Modified `get-shit-done/bin/lib/phase.cjs` — Milestone-scoped phase operations
- Modified `get-shit-done/bin/lib/roadmap.cjs` — Milestone-specific roadmap parsing

### Key Links
- `init execute-phase M7/01` → Returns milestone-scoped phase_dir
- `init plan-phase M7/02` → Creates plan in milestone's phases directory
- `roadmap analyze --milestone M7` → Parses M7's ROADMAP.md
- `progress --milestone M7` → Shows M7's progress only

### Truths
1. `init execute-phase M7/01` returns `phase_dir` under milestones path
2. `progress` without args shows all milestones in multi-bar format
3. Phase operations in milestone context don't affect other milestones
4. Legacy single-milestone projects work unchanged

### Task 3: Update roadmap.cjs for milestone-specific parsing

**What:** Modify roadmap analysis to work with milestone-scoped ROADMAP.md files

**Changes:**
- `cmdRoadmapAnalyze(cwd, milestoneId?)` — Parse specific milestone's roadmap
- `cmdRoadmapGetPhase(cwd, phase, milestoneId?)` — Extract from milestone roadmap
- `cmdRoadmapUpdatePlanProgress(cwd, phase, milestoneId?)` — Update milestone roadmap

**New function:**
```javascript
/**
 * Get path to ROADMAP.md for milestone or project root
 */
function getRoadmapPath(cwd, milestoneId = null) {
  if (milestoneId) {
    const milestonePath = getMilestonePath(cwd, milestoneId);
    return path.join(milestonePath, 'ROADMAP.md');
  }
  return path.join(cwd, '.planning', 'ROADMAP.md');
}
```

**Verification:**
- `roadmap analyze --milestone M7` parses M7's ROADMAP.md
- `roadmap get-phase M7/01` returns phase from M7's roadmap
- Legacy roadmap commands work unchanged
