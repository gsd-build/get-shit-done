# Plan 10-01: Milestone Directory Structure & Core Commands

## Objective

Implement the foundational milestone directory structure and core CLI commands that enable parallel milestone execution. This establishes the file organization pattern and basic CRUD operations for milestones.

## Context

**Source Spec:** `.planning/specs/GSD-PARALLEL-MILESTONES-SPEC.md`

**Current State:**
- GSD assumes single active milestone with phases in `.planning/phases/`
- No support for milestone-scoped phase numbering
- Commands assume flat phase structure

**Target State:**
- Milestones stored in `.planning/milestones/{id}-{slug}/`
- Each milestone has own `ROADMAP.md`, `REQUIREMENTS.md`, and `phases/`
- CLI commands support `milestone/phase` syntax (e.g., `M7/01`)

## Must-Haves

### Artifacts (files that MUST exist after execution)
- `get-shit-done/bin/lib/milestone-parallel.cjs` — Core milestone management functions
- Modified `get-shit-done/bin/gsd-tools.cjs` — New milestone commands routed

### Key Links (code connections that MUST work)
- `gsd-tools milestone create M7 "Patient Engagement"` → creates directory structure
- `gsd-tools milestone list` → shows all milestones with status
- `gsd-tools milestone switch M7` → sets default milestone context
- `gsd-tools find-phase M7/01` → resolves milestone-scoped phase path

### Truths (statements that MUST be verifiable)
1. Creating milestone `M7` produces `.planning/milestones/M7-patient-engagement/` with ROADMAP.md
2. `milestone list` returns JSON array of all milestones with id, name, status, phase_count
3. Existing `.planning/phases/` structure continues to work (backward compatible)
4. `find-phase M7/01` returns correct path under milestones directory

### Task 3: Implement listMilestones function

**What:** Scan `.planning/milestones/` and return metadata for each

**Logic:**
1. Check if `.planning/milestones/` exists
2. List directories matching pattern `{id}-{slug}`
3. For each, read ROADMAP.md to get phase count and completion status
4. Return sorted array by ID

```javascript
    completed_count: 3,
    progress_percent: 60
  }
]
```

**Verification:**
- Returns empty array if no milestones directory

### Task 5: Extend find-phase to support milestone scope

**What:** Modify `cmdFindPhase` to resolve milestone-scoped phase references

**Current behavior:**
- `find-phase 01` → `.planning/phases/01-*/`

**New behavior:**
- `find-phase 01` → Check default milestone, then legacy path
- `find-phase M7/01` → `.planning/milestones/M7-*/phases/01-*/`
- `find-phase --milestone M7 01` → Same as above

**Logic:**
1. Parse reference with `parseMilestonePhaseRef()`
2. If milestone specified, search under that milestone
3. If no milestone but default set, use default
4. Fall back to legacy `.planning/phases/`

**Verification:**
- Legacy `find-phase 01` still works
- `find-phase M7/01` finds phase under milestone
- Returns error if milestone doesn't exist
