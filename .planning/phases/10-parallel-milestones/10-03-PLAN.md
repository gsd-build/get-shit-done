# Plan 10-03: Multi-Milestone State Tracking

## Objective

Extend STATE.md to track progress across multiple parallel milestones. Each milestone has its own progress metrics while global state shows overall project health.

## Context

**Current STATE.md structure:**
```markdown
## Position
phase: 3
plan: 2

## Progress
[██████░░░░] 60%
```

**Target STATE.md structure:**
```markdown
## Milestones

| ID | Name | Status | Progress | Current |
|----|------|--------|----------|---------|
| M1 | Ingestion | active | 80% | Phase 3/5 |
| M7 | Patient Eng | active | 40% | Phase 2/4 |

## Position
milestone: M1
phase: 3
plan: 2

## Progress
Overall: [██████░░░░] 60%
```

## Must-Haves

### Artifacts
- Modified `get-shit-done/bin/lib/state.cjs` — Multi-milestone state handling
- Modified STATE.md schema with milestones section

### Key Links
- `state-snapshot` → Returns milestones array with per-milestone progress
- `state update milestone M7` → Updates M7-specific state
- `state get milestones` → Returns milestone status table

### Truths
1. STATE.md contains `## Milestones` section with all active milestones
2. `state-snapshot --raw` includes `milestones[]` array
3. Switching milestone context updates `## Position` section
4. Legacy STATE.md without milestones section still works

## Tasks

### Task 1: Define multi-milestone STATE.md schema

**What:** Create new STATE.md sections for parallel milestone tracking

**New sections:**

```markdown
---
project: Nova Res
version: 2.0
parallel_milestones: true
active_milestones: [M1, M7]
---

## Milestones

| ID | Name | Status | Progress | Current Phase | Blockers |
|----|------|--------|----------|---------------|----------|
| M1 | Full PubMed Ingestion | active | 80% | 3-bulk-ingestion | |
| M2 | RAG Quality | blocked | 20% | 1-query-transform | Waiting M1 |
| M7 | Patient Engagement | active | 40% | 2-compliance | |

## Position

milestone: M1
phase: 3
plan: 2
status: Executing plan 3-02

## Progress

Overall: [██████░░░░] 60%  (3/5 milestones progressing)

## Recent Activity

- 2026-02-27 14:30 — M7: Completed phase 1-fhir-foundation
- 2026-02-27 12:15 — M1: Started phase 3-bulk-ingestion
```

**YAML frontmatter additions:**
- `parallel_milestones: true` — Enable parallel mode
- `active_milestones: [M1, M7]` — Quick reference

**Verification:**
- Schema documented in state.cjs comments
- Template created for new parallel projects

### Task 4: Implement cross-milestone activity logging

**What:** Track recent activity across all milestones in STATE.md

**Activity log format:**
```markdown
## Recent Activity

- 2026-02-27 14:30 — M7: Completed phase 1-fhir-foundation
- 2026-02-27 12:15 — M1: Started phase 3-bulk-ingestion
- 2026-02-27 10:00 — M1: Completed plan 2-03
```

**Implementation:**
- New `cmdStateLogActivity(cwd, milestone, activity)` function
- Called by plan execution, phase completion
- Maintains last 20 entries (configurable)
- Timestamp in ISO format

**Activity types:**
- Phase started/completed
- Plan started/completed
- Blocker added/resolved
- Milestone status changed

**Verification:**
- Activity logged with milestone prefix
- Log truncated at max entries
- Timestamps are accurate
