# Plan 10-04: Migration Tool & Backward Compatibility

## Objective

Ensure existing GSD projects can seamlessly upgrade to parallel milestones, and provide a migration tool for projects wanting to restructure existing phases into multiple milestones.

## Context

**Migration scenarios:**
1. **Legacy project, no change** — Continue with `.planning/phases/`, no migration needed
2. **Legacy project, add milestones** — Create new milestones alongside existing phases
3. **Legacy project, full migration** — Move existing phases into milestone structure
4. **New project** — Start fresh with parallel milestones

## Must-Haves

### Artifacts
- `get-shit-done/bin/lib/migrate.cjs` — Migration functions
- New command: `gsd-tools migrate-to-parallel`

### Key Links
- `gsd-tools migrate-to-parallel --dry-run` → Shows what would change
- `gsd-tools migrate-to-parallel --execute` → Performs migration
- Legacy commands continue working without modification

### Truths
1. Existing projects with `.planning/phases/` work without any changes
2. Migration tool creates backup before modifying structure
3. Mixed mode (legacy phases + milestones/) supported during transition
4. All existing tests pass after migration implementation

## Tasks

### Task 1: Implement backward compatibility layer

**What:** Ensure all commands check for parallel mode and fall back gracefully

**Compatibility rules:**
1. If `.planning/milestones/` doesn't exist → legacy mode
2. If `config.json` has `parallel_milestones: false` → legacy mode
3. Legacy mode uses `.planning/phases/` directly
4. No errors or warnings for legacy projects

**Functions to wrap:**
```javascript
/**
 * Get phases directory, respecting project mode
 */
function getProjectPhasesDir(cwd, milestoneId = null) {
  if (milestoneId || isParallelMilestoneProject(cwd)) {
    return getMilestonePhasesDir(cwd, milestoneId);
  }
  return path.join(cwd, '.planning', 'phases');
}

/**
 * Get ROADMAP.md path, respecting project mode
 */
function getProjectRoadmapPath(cwd, milestoneId = null) {
  if (milestoneId || isParallelMilestoneProject(cwd)) {
    return getMilestoneRoadmapPath(cwd, milestoneId);
  }
  return path.join(cwd, '.planning', 'ROADMAP.md');
}
```

**Verification:**
- All existing tests pass without modification
- Legacy project commands work unchanged
- No "milestone" references leak into legacy output

### Task 4: Add migration command to gsd-tools.cjs

**What:** Route migration commands through CLI

**Commands:**
```bash
migrate-to-parallel                    # Interactive wizard
migrate-to-parallel --dry-run          # Preview only
migrate-to-parallel --auto             # Auto-detect boundaries
migrate-to-parallel --config <file>    # Use mapping file
migrate-to-parallel --restore <backup> # Restore from backup
```

**Backup structure:**
```
.planning/backups/
└── pre-migration-2026-02-27-143000/
    ├── manifest.json       # What was backed up
    ├── phases/             # Original phases
    ├── ROADMAP.md          # Original roadmap
    └── STATE.md            # Original state
```

**Verification:**
- `migrate-to-parallel --dry-run` shows actions
- `migrate-to-parallel` creates backup before changes
- `--restore` successfully reverts migration

## Deviation Handling

| Deviation | Action |
|-----------|--------|
| Migration fails mid-execution | Auto-restore from backup |
| Disk space insufficient for backup | Warn and abort before starting |
| Phase references in PLAN.md break | Update relative paths during migration |
| User cancels mid-wizard | No changes made |
