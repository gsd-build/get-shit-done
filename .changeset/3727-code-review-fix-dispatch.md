**`/gsd-code-review N --fix [--all] [--auto]` now actually dispatches to the autofix workflow** — `code-review.md`'s `initialize` step previously parsed only `--depth` and `--files`; `--fix` / `--all` / `--auto` (advertised in the `argument-hint` since the #2790 consolidation merged the deleted `/gsd-code-review-fix` standalone) were silently dropped. `autonomous.md`'s `Skill(skill="gsd-code-review", args="${PHASE} --fix --auto")` and `execute-phase.md`'s `--fix` hint both silently no-op'd the autofix half of the loop. PR #2947's title/body described this exact dispatch wire-up, but the merged diff only touched `phase.cjs` + a phase test — the workflow change was never shipped. Fixed by (1) parsing `FIX_FLAG` / `FIX_ALL` / `AUTO_MODE` from `$ARGUMENTS` in `code-review.md`'s `initialize`, (2) adding a `dispatch_fix` step after `present_results` that forwards `${PHASE_ARG} [--all] [--auto]` into the existing `code-review-fix.md` workflow when `FIX_FLAG=true` and STATUS ≠ "clean", and (3) preloading `code-review-fix.md` in `commands/gsd/code-review.md`'s `<execution_context>` so the dispatch hands off in-context (no extra Read at runtime). Regression covered by `tests/bug-3727-code-review-fix-dispatch.test.cjs` (12 assertions across initialize-step flag parsing, dispatch_fix step shape, workflow order, and execution_context preload).
