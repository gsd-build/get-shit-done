**Installed agent bodies no longer ship retired `/gsd:<cmd>` colon references for hyphen-name runtimes** — companion to the #3583 SKILL.md fix. The Claude/default (and Qwen/Hermes) branch of the agent write loop in `bin/install.js` copied `agents/*.md` bodies verbatim, so canonical source colon references (required by the #3443 invariant) survived into `~/.claude/agents/gsd-*.md`. Since #2808 all GSD skills/agents register under the hyphen `name:` form, so an agent that surfaces an instruction like "re-run `/gsd:plan-phase --research-phase <N>`" (e.g. `gsd-executor.md`) hands the user a command Claude Code rejects with `Unknown command: /gsd:plan-phase. Did you mean /gsd-plan-phase?`. The agent loop now applies the shared `transformColonToHyphen` (new inverse export in `scripts/fix-slash-commands.cjs`, roster-checked, word-boundary safe) for the same Claude/Qwen/Hermes set the SKILL.md fix targets. Copilot/Codex/etc. are excluded (their `convertClaudeAgentTo*` adapters already convert command namespaces) and Gemini is excluded intentionally (its slash commands are colon-namespaced). Empirically: a sandboxed full Claude install drops from 28 agent files / 96 colon refs to 0. Regression covered by `tests/bug-3583-claude-agent-body-transform.test.cjs`. Extends #3583.
