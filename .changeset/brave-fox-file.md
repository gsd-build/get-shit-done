**`/gsd-feedback`** — new command for user-driven issue intake (bugs, feature requests, questions) without leaving a session. Complements `/gsd-forensics` (investigation-driven); this command is proactive. Files via `gh issue create`, prefilled browser URL, or copy-paste fallback. For bugs, optionally attaches diagnostics from `/gsd-forensics`.
