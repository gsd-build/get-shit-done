# Beta-фичи GSD

> **Beta-фичи — opt-in и могут измениться или быть удалены без предупреждения.** Они не покрываются гарантиями стабильного API, действующими для остальной части GSD. Если beta-фича переходит в stable, это будет задокументировано в [COMMANDS.md](COMMANDS.md) и [FEATURES.md](FEATURES.md) с записью в changelog.

## `/gsd-ultraplan-phase` — Интеграция Ultraplan [BETA]

> **Только Claude Code · Требуется Claude Code v2.1.91+**
> Ultraplan сам по себе является research preview Claude Code — и эта команда, и сама фича могут меняться.

### Что делает

`/gsd-ultraplan-phase` выгружает черновую разработку plan-фазы GSD в облачную инфраструктуру [Claude Code ultraplan](https://code.claude.ai). Вместо локального планирования в терминале план рисуется в браузерной сессии с:

- **Боковой панелью outline** для навигации по структуре плана
- **Inline-комментариями** для аннотирования и доработки задач
- Постоянной вкладкой браузера, чтобы твой терминал оставался свободным, пока план разрабатывается

Когда черновик тебя устраивает, ты сохраняешь его и импортируешь обратно в GSD — детекция конфликтов, валидация формата и проверка plan-checker'ом запускаются автоматически.

### Зачем использовать

| Ситуация | Рекомендация |
|----------|--------------|
| Длинные сложные фазы, план которых хочется прочитать и прокомментировать до исполнения | Используй `/gsd-ultraplan-phase` |
| Быстрые фазы, знакомый домен или рантаймы не-Claude Code | Используй `/gsd-plan-phase` (стабильный) |
| У тебя есть план из другого источника (коллега, внешний AI) | Используй `/gsd-import` |

### Требования

- **Рантайм:** Только Claude Code. Команда падает с ошибкой на Gemini CLI, Copilot CLI и других рантаймах.
- **Версия:** Claude Code v2.1.91 или выше (переменная окружения `$CLAUDE_CODE_VERSION` должна быть установлена).
- **Стоимость:** Без доплат для подписчиков Pro и Max. Ultraplan включён без дополнительной платы.

### Использование

```bash
/gsd-ultraplan-phase         # Ultraplan для следующей незапланированной фазы
/gsd-ultraplan-phase 2       # Ultraplan для конкретного номера фазы
```

| Аргумент | Обязательный | Описание |
|----------|--------------|----------|
| `N` | Нет | Номер фазы (по умолчанию — следующая незапланированная) |

### Как это работает

1. **Инициализация** — GSD запускает стандартный init plan-фазы, определяя какую фазу планировать и подтверждая предусловия.

2. **Сборка контекста** — GSD читает `ROADMAP.md`, `REQUIREMENTS.md` и любой существующий `RESEARCH.md` для фазы. Этот контекст упаковывается в структурированный промпт, чтобы у ultraplan было всё необходимое и тебе не пришлось ничего копировать вручную.

3. **Инструкции возврата** — Перед запуском ultraplan GSD печатает команду импорта в твой терминал, чтобы она была видна в scroll-back буфере после окончания браузерной сессии:
   ```
   When done: /gsd-import --from <path-to-saved-plan>
   ```

4. **Запуск Ultraplan** — Команда `/ultraplan` передаёт управление браузеру. Используй sidebar outline и inline-комментарии, чтобы пересмотреть и доработать черновик.

5. **Сохрани план** — Когда доволен результатом, нажми **Cancel** в Claude Code. Claude Code сохранит план в локальный файл и вернёт тебя в терминал.

6. **Импорт обратно в GSD** — Запусти команду импорта, напечатанную на шаге 3:
   ```bash
   /gsd-import --from /path/to/saved-plan.md
   ```
   Это запускает детекцию конфликтов с `PROJECT.md`, конвертирует план в формат GSD, валидирует его через `gsd-plan-checker`, обновляет `ROADMAP.md` и коммитит — тот же путь, что и для любого внешнего импорта плана.

### Что получается на выходе

| Шаг | Результат |
|-----|-----------|
| После ultraplan | Внешний файл плана (сохранённый Claude Code) |
| После `/gsd-import` | `{phase}-{N}-PLAN.md` в `.planning/phases/` |

### Чего эта команда НЕ делает

- Не пишет файлы `PLAN.md` напрямую — все записи идут через `/gsd-import`
- Не заменяет `/gsd-plan-phase` — локальное планирование никак не затрагивается и остаётся по умолчанию
- Не запускает агентов исследования — если тебе нужен `RESEARCH.md`, запусти сначала `/gsd-plan-phase --skip-verify` или research-only проход

### Решение проблем

**«ultraplan is not available in this runtime»**
Ты запустил GSD вне Claude Code. Перейди в терминальную сессию Claude Code или используй `/gsd-plan-phase`.

**Браузерная сессия ultraplan не открылась**
Проверь версию Claude Code: `claude --version`. Требуется v2.1.91+. Обнови через `claude update`.

**`/gsd-import` сообщает о конфликтах**
Ultraplan мог предложить что-то, противоречащее решению в `PROJECT.md`. Шаг импорта попросит разрешить каждый конфликт прежде, чем что-либо записать.

**Plan checker падает после импорта**
В импортированном плане есть структурные проблемы. Посмотри вывод проверяющего, отредактируй сохранённый файл, чтобы их исправить, и перезапусти `/gsd-import --from <тот-же-файл>`.

### Связанные команды

- [`/gsd-plan-phase`](COMMANDS.md#gsd-plan-phase) — стандартное локальное планирование (стабильное, все рантаймы)
- [`/gsd-import`](COMMANDS.md#gsd-import) — импортировать любой внешний файл плана в GSD
