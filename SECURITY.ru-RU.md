# Политика безопасности

## Сообщение об уязвимости

**Пожалуйста, не сообщайте об уязвимостях через публичные GitHub Issues.**

Вместо этого пришлите письмо на: **security@gsd.build** (или напишите в DM @glittercowboy в Discord/Twitter, если письмо не доходит)

Укажите:
- Описание уязвимости
- Шаги воспроизведения
- Потенциальное влияние
- Предложения по исправлению (опционально)

## Сроки ответа

- **Подтверждение получения**: в течение 48 часов
- **Первичная оценка**: в течение 1 недели
- **Срок исправления**: зависит от серьёзности, но мы целимся в:
  - Critical: 24–48 часов
  - High: 1 неделя
  - Medium/Low: следующий релиз

## Область

Проблемы безопасности в кодовой базе GSD, которые могут:
- Выполнить произвольный код на машине пользователя
- Раскрыть чувствительные данные (API-ключи, креды)
- Скомпрометировать целостность сгенерированных планов/кода

## Признание

Мы ценим ответственное раскрытие и упомянем репортёров в release notes (если не предпочитаете остаться анонимом).
